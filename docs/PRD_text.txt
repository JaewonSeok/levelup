레벨업(승진) 관리 시스템
3단계 PRD (Product Requirements Document)
Confidential — Internal Use Only
1. 문서 개요
1.1 목적
본 문서는 사내 직원 레벨업(승진) 관리 웹 프로그램의 기능 요구사항을 정의한다. 기존 CUBE 시스템의 레벨업관리 모듈을 참고하여, 각 본부/팀별 승진 대상자 데이터를 엑셀 파일로 업로드하고, 포인트/학점 관리, 대상자 선정, 심사, 확정까지의 전체 승진 프로세스를 3단계로 나누어 개발한다.
1.2 용어 정의
1.3 전체 프로세스 흐름
승진 관리는 다음 순서로 진행된다:
데이터 준비: 본부/팀별 승진 대상자 엑셀 파일 업로드 및 기초 데이터 등록
포인트/학점 관리: 연도별 포인트 및 학점 데이터 관리, 충족 여부 자동 판정
대상자 선정: 포인트/학점 기준 충족 직원을 대상자로 선정, 인사팀 심사대상 체크
심사: 소속/타본부 본부장 의견 수렴 및 역량레벨평가 수행, 추천/미추천 결정
확정: 대표이사 최종 승진 확정/보류 결정, 확정률 집계
1.4 개발 단계 요약
2. Phase 1 — 기반 구축 및 데이터 관리
Phase 1은 시스템의 기반이 되는 데이터 구조 설계, 엑셀 업로드를 통한 초기 데이터 적재, 그리고 레벨/포인트/학점의 기본 조회 및 관리 기능을 구현한다.
2.1 엑셀 파일 업로드
2.1.1 기능 개요
각 본부/팀별 승진 대상자 데이터를 엑셀(.xlsx) 파일로 업로드하면, 시스템이 데이터를 파싱하여 DB에 저장하고 즉시 화면에 반영한다. 엑셀 파일은 사전에 정의된 템플릿 양식을 따라야 하며, 업로드 시 유효성 검증을 수행한다.
2.1.2 엑셀 템플릿 컬럼 정의
2.1.3 업로드 요구사항
2.2 레벨 관리/조회
직원의 기본 인사정보와 현재 직급(레벨) 정보를 조회하는 화면이다. 기본 조회(검색 영역 1)와 상세 조회(검색 영역 2)의 2단계 검색을 지원한다.
2.2.1 화면 구성
2.3 레벨업 포인트 관리
직원별 연도별 승진 포인트를 관리하는 화면이다. 해당 레벨의 연차만큼만 포인트를 출력하며, 상점/벌점이 해당 레벨 체류 기간에만 집계된다.
2.3.1 요구사항
2.4 레벨업 학점 관리
직원별 연도별 학점을 관리하는 화면이다. 포인트와 동일한 2단계 조회 구조를 가지며, 입사일 컬럼이 추가된다.
2.5 Phase 1 데이터 모델 (주요 엔티티)
3. Phase 2 — 심사 프로세스
Phase 2는 포인트/학점 충족 여부에 따른 대상자 자동 선정, 인사팀의 심사대상 지정, 본부장 의견 수렴, 역량레벨평가 수행, 그리고 추천/미추천 결정까지의 심사 워크플로우를 구현한다.
3.1 레벨업 대상자 관리
포인트 또는 학점 또는 포인트&학점을 충족한 직원만을 대상자로 자동 필터링하여 출력하고, 인사팀 담당자가 심사대상 여부를 체크하여 저장하는 화면이다.
3.1.1 요구사항
3.2 레벨업 심사
심사대상으로 선정된 직원에 대해 역량레벨평가, 본부장 의견 수렴, 추천/미추천 결정을 수행하는 핵심 심사 화면이다. 본인 소속 대상자와 타본부 소속 대상자를 모두 평가할 수 있다.
3.2.1 심사 메인 화면
3.2.2 본부장 의견 입력 팝업
의견란의 미입력(경고 아이콘) 클릭 시 표시되는 팝업으로, 대상자의 요약 정보와 각 본부장의 의견 입력란이 표시된다.
3.2.3 심사 요구사항
3.3 Phase 2 데이터 모델 (추가 엔티티)
4. Phase 3 — 확정 및 고도화
Phase 3는 대표이사의 최종 승진 확정, 통계 대시보드, 연도별 이력 관리, 그리고 시스템 전반의 권한 고도화를 구현한다.
4.1 레벨업 확정
레벨업 대상자를 본부별/팀별/레벨별로 조회하고, 본부장으로부터 추천을 받은 대상, 추천을 받지 못한 대상, 전체를 조회할 수 있으며 대표이사가 최종 확정/보류를 결정하는 화면이다.
4.1.1 요구사항
4.2 레벨업 기준 설정
레벨별 승격 기준 포인트, 학점, 체류 연수 등의 기준값을 관리하는 시스템 설정 화면이다.
